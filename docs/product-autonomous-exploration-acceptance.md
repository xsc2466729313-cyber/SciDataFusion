# Topic-first autonomous exploration acceptance

## Definition of Done

- [x] The run form requires only a research direction; evidence focus is optional and collapsed.
- [x] A strict Qwen plan contains a topic profile and one to four complementary search expressions.
- [x] The profile covers evidence priorities, multi-source types, candidate fields, quality checks,
  target outputs, and a visualization direction.
- [x] A missing or invalid model plan falls back to a deterministic topic-derived profile and query.
- [x] A request containing only `execution_mode` and `research_goal` completes through the API.
- [x] Online workbench output removes built-in reference fields, artifacts, scientific values,
  evidence rows, quality gates, retrieval hits, and formal delivery availability.
- [x] The overview blueprint, metrics, process stages, coverage chart, candidate fields, quality plan,
  and interactive graph are derived from the current research topic.
- [x] Real search results appear as untrusted discovery nodes and retain their source URL.
- [x] Candidate fields remain plans with empty values until real source material is parsed.
- [x] Offline mode remains a clearly labeled reproducible reference demonstration.
- [x] Ruff, mypy, pytest, Bandit, secret scan, dependency audit, and browser checks pass.
