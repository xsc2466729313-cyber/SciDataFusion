# ADR 0036: 本机配置与中文证据图谱

## 状态

Accepted for M29.

## 决策

联网任务提交前必须读取脱敏配置状态。配置不完整时不创建后台任务，而是引导用户进入“联网配置”。运行中发生的异常仍写入服务端日志，但任务记录只保存稳定错误码、中文说明和恢复动作，不保存密钥、外部原文或未经校验的异常详情。

Docker 工作台只绑定 `127.0.0.1`。Nginx 覆盖写入内部代理标记，API 仅接受回环请求，或来自私有网络且带该标记的请求。配置写入 `/app/config/.env` 对应的持久卷；GET 接口只返回是否已配置，绝不返回密钥。

图谱底层标识和英文枚举保持稳定，中文只作为展示语义。当前主题结构化数据增加以下有向边：

| 关系 | 起点 | 终点 | 含义 |
| --- | --- | --- | --- |
| 数据集包含证据 | 数据集 | 单元格证据 | 该证据从此内容寻址数据集解析得到 |
| 证据支持字段 | 单元格证据 | 原始字段 | 该证据位于此列，不代表语义映射已确认 |
| 同一记录共现 | 单元格证据 | 相邻单元格证据 | 两个证据来自同一原始记录，不代表因果或一致 |
| 映射到目标字段 | 原始字段 | 目标字段 | 通过确定性或受控 AI 字段映射提出，可复核 |

## 后果

用户可以从同一中文工作台完成配置、运行和故障恢复。图谱能够说明数据、证据与字段的来源关系，同时不把视觉连边误解为科学结论。代理信任依赖 API 不对宿主机公开，Compose 的回环绑定不得删除。
